\textbf{Executive Summary.}
The rollout-side oracle-conditioning bug was fixed, regression tests were added, the TeacherOracle subgoal model was rerun, and the fixed evaluator was sanity-checked with the direct-act oracle action model. The final TeacherOracle result remains a decisive Gate A failure: \textbf{0.0\% world success on execute-1 and 0.0\% world success on execute-2}. The Push-T continuous-subgoal branch stops at Gate A.

\textbf{Experiment ID.} \texttt{pusht\_subgoal\_distill\_round2\_oraclefix}

\section*{1. Model Pipeline}

\begin{figure}[h]
  \centering
  \includegraphics[width=\textwidth]{model_pipeline.pdf}
  \caption{Continuous-subgoal pipeline used by the final Push-T oracle-fix rerun. The TeacherOracle path encodes a step-aligned future window at every rollout step.}
\end{figure}

The model uses a shared action decoder with two subgoal sources:
\begin{itemize}
  \item \textbf{TeacherOracle path}: \texttt{FutureBottleneck} encodes a future rollout window into a continuous subgoal.
  \item \textbf{Student path}: \texttt{SubgoalPredictor} predicts the continuous subgoal from encoded history.
\end{itemize}

The final rerun evaluates the TeacherOracle path. StudentFrozen and StudentJoint are intentionally skipped after Gate A fails.

\section*{2. Experiment Flow}

\begin{figure}[h]
  \centering
  \includegraphics[width=\textwidth]{experiment_flow.pdf}
  \caption{Final experiment flow from oracle-conditioning fix to Gate A hard stop.}
\end{figure}

Gate A requires TeacherOracle success of at least 90\% on execute-1 and execute-2 over 50 rollouts with 100 world steps per rollout. The final rerun uses \texttt{eval.rollout\_batch\_size=10} for stable local RTX 4070 8 GB evaluation. This changes batching only, not evaluation semantics.

\section*{3. Implementation Fixes}
\begin{itemize}
  \item \textbf{Stepwise oracle conditioning.} \texttt{DirectActPolicy} and \texttt{SubgoalPolicy} consume per-step oracle plan/subgoal tensors and advance a rollout-step counter during world evaluation.
  \item \textbf{TeacherOracle world evaluation.} \texttt{eval\_subgoal\_policy.py} generates oracle subgoals from rollout windows aligned to the current rollout step.
  \item \textbf{Direct-act oracle evaluation.} \texttt{eval\_policy.py} advances oracle plan conditioning by rollout step and uses the configured maximum evaluation length.
  \item \textbf{Regression coverage.} Unit tests verify stepwise oracle plan selection and stepwise oracle subgoal selection.
\end{itemize}
